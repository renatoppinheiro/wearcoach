# Plan 01-02: Data Fetch & Coaching Chat API Integration

<objective>
Implement `/api/fetch` and `/api/chat` API endpoints in `coach/server.py` to allow triggering data pulls and conducting LLM coaching chat sessions over HTTP REST endpoints.
</objective>

<tasks>
<task id="1" type="tracer">
<name>Implement POST /api/fetch endpoint</name>
<files>coach/server.py</files>
<action>
Add `POST /api/fetch` endpoint to `coach/server.py`.
Calls `_fetch_all(days)` from `main.py` / `coach` modules, saves snapshot to `data/snapshot-YYYY-MM-DD.json`, and returns JSON response with activity & wellness count.
</action>
<verify>
Run unit tests in `tests/test_server.py`.
</verify>
</task>

<task id="2" type="tracer">
<name>Implement POST /api/chat endpoint with chart JSON block extraction</name>
<files>coach/server.py</files>
<action>
Add `POST /api/chat` endpoint taking `{"message": "..."}`.
Loads latest snapshot data and calls `coach/llm.py` via `coach/prompt.py`.
Extracts any ````chart ... ```` JSON blocks from the LLM text response and returns `{"reply": "cleaned text", "chart": {...}}`.
</action>
<verify>
Run unit tests in `tests/test_server.py`.
</verify>
</task>

<task id="3" type="tracer">
<name>Expand unit test suite for fetch and chat endpoints</name>
<files>tests/test_server.py</files>
<action>
Add test cases in `tests/test_server.py` using `unittest.mock` to test:
- `POST /api/fetch` returns 200 OK and snapshot summary
- `POST /api/chat` handles prompt calls and parses ````chart```` JSON blocks accurately
</action>
<verify>
Run `pytest tests/test_server.py`.
</verify>
</task>
</tasks>
